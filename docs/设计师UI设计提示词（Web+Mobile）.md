你是一位资深产品与视觉设计师，请基于以下信息完成 Web 端的信息架构（IA）与界面设计（Mobile 暂不做）。

【项目背景】
- 这是一个围绕“北邮人论坛镜像行为”构建的论坛产品。
- 产品中有两类核心参与者：真实用户 + 机器人（机器人是一等参与者，不是辅助功能）。
- 前端会周期性拉取后端数据，并将其转化为机器人发帖/回帖行为。
- 匿名通知是核心能力：用户可通过绑定当天机器人收件箱接收通知。
- 未来可能支持：用户直接认领某个镜像帖子/回复，并订阅变更通知。
- 当前不预设“真实用户可直接发帖”。
- 主要语言为中文

【页面结构（从上到下）】
首页请按以下顺序组织：
1. 顶部搜索框
   - 支持搜索帖子 / 回复 / 用户
2. 信息流
   - 机器人信息流
   - 真实用户信息流
3. 所有分区与子分区（论坛导航区）

【关键产品约束】
- 绑定机器人按钮：放在“机器人主页”中，不放在首页。
- 订阅帖子 / 订阅回复：放在“帖子详情页”中。
- Web 端列表交互采用分页，不使用无限滚动。
- Mobile 暂不设计。

【请先完成 IA（必须先做）】
请按以下结构输出：

1. 全局布局（Web）
   - 建议采用“左侧导航 + 右侧主内容区”的工作台结构
   - 左侧：分区树、快捷入口、站点导航
   - 主区：搜索、信息流、列表、分页

2. 导航结构（Web）
   - 一级导航项（MVP 必须出现）
   - 二级导航项（可折叠）
   - 预留导航项（未来功能占位）

【页面清单与每页内容要求（请逐页展开）】
请至少覆盖以下页面，并为每个页面给出：
- 页面目标
- 核心模块（区块）
- 每个模块的字段/信息项
- 关键操作（按钮、入口）
- 页面状态（空、加载中、失败、无权限/未绑定等）
- 分页规则（每页条数建议、翻页交互、总数展示）

必做页面如下：

A. 首页 / 时间流（MVP）
- 展示内容：顶部搜索入口 + 机器人信息流 + 真实用户信息流 + 分区/子分区导航
- 必须包含：分页, 按最新回复还是最新发布排序的导航. 每个帖子列表包括:帖子标题, 发布时间、帖子发布者, 最后回复者, 回复数, 人物头像, 全文字内容, 没有图片
- 模块联动要求：鼠标点击不同模块时，主内容区切换为对应帖子列表
  - 点击顶部搜索框后：弹出悬浮搜索窗口（非跳转）
  - 在悬浮搜索窗口中提交关键词后：主内容区切换到搜索结果帖子列表（后续可扩展到回复/用户结果）
  - 点击机器人信息流：显示机器人相关帖子列表（分页）
  - 点击真实用户信息流：显示真实用户相关帖子列表（分页）
  - 点击分区/子分区：显示该分区下帖子列表（分页）
- 建议模块：
  - 顶部搜索框（占位：搜索帖子/回复/用户）
  - 机器人信息流列表（分页）
  - 真实用户信息流列表（分页）
  - 分区与子分区导航区（树状/可折叠）

B. 帖子详情页（MVP）
- 展示内容：主帖、回复楼层
- 必须包含：镜像来源信息、内容失效提示（源站删除场景）、机器人行为标记
- 关键约束：订阅帖子、订阅回复功能只放本页
- 建议模块：
  - 主帖区
  - 回复列表区（分页）
  - 订阅操作区（订阅该帖 / 订阅某条回复）

C. 机器人主页（MVP）
- 展示内容：机器人身份、职责说明、近期行为
- 关键约束：绑定机器人按钮只放本页
- 建议模块：
  - 机器人基础信息
  - 近期发帖/回帖记录（分页）
  - 绑定状态卡片
  - 绑定操作区（绑定、校验、结果反馈、失败重试）

D. 通知中心（MVP）
- 展示内容：匿名通知、帖子变化提醒、系统消息
- 必须包含：未读状态、通知类型、关联对象（帖子/回复）、时间
- 建议模块：
  - 通知分类
  - 通知列表（分页）
  - 快捷操作（已读、批量已读、跳转原文）

E. 分区页 / 子分区页（MVP）
- 展示内容：该分区下帖子列表、子分区切换
- 必须包含：分区说明、帖子排序、分页
- 建议模块：
  - 分区头部信息
  - 子分区切换
  - 帖子列表（分页）

F. 个人中心（MVP）
- 展示内容：通知偏好、绑定状态摘要、订阅摘要
- 必须包含：账户状态（匿名/已绑定）、通知设置、订阅管理入口
