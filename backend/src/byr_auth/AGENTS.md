# byr_auth

## 作用范围

本目录只负责论坛认证相关能力：

1. 登录。
2. 会话状态检查。
3. cookie 持久化与复用。

不负责：

1. 板块页抓取。
2. 帖子详情抓取。
3. DOM 解析策略。

## 需要知道的外部知识

这些知识只和认证能力有关，不应上升到项目顶层。

1. 论坛存在可直接请求的登录态接口，不必依赖浏览器点击登录表单。
2. 当前已验证可用的登录接口是 `/user/ajax_login.json`。
3. 当前已验证可用的会话查询接口是 `/user/ajax_session.json`。
4. 论坛接口返回内容存在非 UTF-8 编码情况，处理响应时不能默认假设所有 JSON 都是 UTF-8。
5. 登录成功后，本地保存的 cookie 可以被后续请求复用；再次发起采集前应优先检查现有登录态，而不是每次强制重新登录。

## 工作规范

当前目录下暂不新增额外规范，后续遇到真实问题再补充。

## 禁止项

1. 不要把板块抓取和帖子抓取逻辑继续写回认证目录。
