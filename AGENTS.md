# 项目级协作说明

## 项目背景

- 这是一个围绕北邮人论坛镜像行为构建的论坛沙盒。
- 前端目标是做成一个完整论坛，逐步具备发帖、回复、展示、通知、登录等能力。
- 沙盒里同时存在两类参与者：真实用户与机器人。
- 机器人不是辅助角色，而是这个产品中的一等参与者，甚至可以说是更重要的参与者。
- 前端会周期性向后端请求北邮人数据，并自动整理、转换、发布为机器人发帖与回复行为。
- 匿名通知是项目的核心目标之一。
- 真实用户通过订阅具体的镜像帖子或镜像回复来接收通知。
- 当前规划不支持“绑定机器人”作为通知范围；机器人 ID 可能对应多个真实源站用户，不能把机器人与帖子/回复订阅混在同一层语义里。
- 后续通知入口应优先支持搜索回复、搜索机器人用户，并在机器人详情页展示最近发帖和最近回复，方便真实用户快速订阅具体帖子或回复。
- 订阅帖子或回复只表达“接收通知”，不表达“认领身份”或“该源站内容属于该真实用户”。
- 是否允许真实用户直接在这个论坛中发帖，目前尚未定案，默认不要擅自假设。
- 北邮人源站历史内容可能消失，旧帖子可能被定期删除，因此不能把源站视为永久可追溯的数据源。
- 正式上线前，预计会从旧的 PostgreSQL + Prisma 流水线迁移历史数据。

## 当前阶段目标

- 先建立稳定、可复用的后端采集能力。
- 先暴露首个面向前端的同步 API，支持主拉取和指定帖子补拉。
- 先明确项目级说明文档边界，让产品背景放在根目录，模块细节贴近实现目录。
- 当前阶段优先跑通“后端采集 -> 前端整理 -> 机器人发帖/通知”的最小链路。

## 文档规则

- 仓库内新增或维护的 Markdown 文档默认使用中文，除非有明确的特殊原因需要使用其他语言。

## 提交规则

- git 提交信息默认使用中文，除非有明确的特殊原因需要使用其他语言。

## 边界原则

- 项目级背景、跨模块规则、产品目标放在仓库根目录。
- 模块专属事实和实现约束放在对应子目录的 `AGENTS.md` 中。
- `backend/AGENTS.md` 只应保留后端需要知道的内容，不应继续承载整个产品背景。
